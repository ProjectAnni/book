# 专辑标签

为了显示更加有序的专辑列表，元数据仓库以 `Tag` 的形式对专辑、唱片和音轨进行了分类。

专辑标签信息通过 `toml` 格式描述。

```toml
######################################################
# Tag 信息
[[tag]]
# Tag 名称 [必填项]
name = "THE IDOLM@STER MILLION LIVE"
#
# Tag 别名 [可选项]
#
# 别名仅作展示用，不起任何指代作用。
alias = []
#
# Tag 类型 [必填项]
#
# 可选值如下所示：
# artist：单艺术家
# group：（多艺术家）组合
# animation：动画
# series：系列
# project：企划
# game：游戏
# organization：组织、社团、公司
# default：默认，未分类
#
# 此外，还有一种特殊的 Tag：
# category：属性，如 OP、ED、OST
# 用于表示专辑/碟片/音轨的具体属性，这类 Tag 不会出现在 Tag 图中
type = "artist"
#
# Tag 的父级 Tag [可选项]
#
# 父 Tag 必须作为已有的 Tag 存在。
included-by = ["THE IDOLM@STER"]
#
# Tag 的子 Tag [可选项]
#
# 指定子 Tag 时会自动创建对应的 Tag。如果该 Tag 已经存在，检查时会抛出一个错误。
#
# 对结构复杂的 Tag 不建议使用子 Tag，建议以 included-by 上指的形式构建依赖关系。
# 子 Tag 可以在 Album 信息中使用。
#
# 子 Tag 根据 includes 中的字段名区分类型。
includes.series = ["THE IDOLM@STER LIVE THE@TER DREAMERS"]
includes.game = ["THE IDOLM@STER Million Live Theater Days"]
```
